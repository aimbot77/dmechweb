[02] How It Works

Five-Layer Architecture

Sensor data flows through a deterministic pipeline: physics, fuse, decide, verify, act — with full audit trail at every stage.

Layer 1
Physics & Environment Engine
ISA Standard Atmosphere (0–86 km): Real air density, temperature, pressure, speed of sound at altitude
Dryden Turbulence Model: Ornstein-Uhlenbeck stochastic process for wind gusts (mean-reverting, exact discretisation)
Weather Modifiers: Rain intensity, fog density, gust magnitude dynamically affect sensor performance
Terrain Engine: Procedural elevation mesh, ray-march line-of-sight checks — system knows when a threat is behind a ridge
Drag Physics: Quaternion 3D geometry + Mach compressibility correction on drag polar
→
Layer 2
Hardware Sensor Stack
RTL-SDR V3: Detects RC control links at 433/915 MHz and 2.4 GHz FHSS/DSSS, signal strength + rough distance estimate, non-blocking
Webcam Visual Tracker: OpenCV background subtraction + contour detection, pixel-to-arena coordinate conversion, velocity via rolling linear regression, <50ms frame budget
Benewake TFMini LiDAR: UART at 100 Hz, checksum validation, kill confirmation when interceptor closes to kill radius
nRF24L01 Backup Comms: Dual-role TX (Pi 4 → interceptors for GOTO/RTB/HOLD when MAVLink is down) and RX (threat drone → Pi 4 for high-accuracy secondary track), 32-byte payload, 10 Hz max
ASTERIX + ADS-B adapters: Standard radar protocols for real radar integration
MAVLink Bridge: Interceptor telemetry and control
→
Layer 3
Decision Engine
CCPL RL Framework: Indigenous — zero foreign ML dependencies, cannot be restricted or backdoored
Hungarian Optimizer: Optimal effector assignment
Threat Prioritization & Asset Allocation: Dynamic real-time scoring
Full Audit Trail: State saved at every tick
→
Layer 4
Safety Layer
Pre-arm Checklist: 6 mandatory checks before any interceptor can arm (safety config, geofence, arena config, kill switch channel, engagement mode, IC separation). System CANNOT proceed if any single check fails.
Geofence Enforcement: Polygon boundary, ray-casting point-in-polygon, stateless per check, <0.1ms latency, fail-safe on uncertainty (blocks command, never guesses)
Hardware Kill Switch: Monitors RC channel 8 via MAVLink, debounced with 3 consecutive readings to prevent RF noise false triggers, immediately RTBs ALL interceptors on activation
Safety Gate: Centralized gate that ALL commands pass through before reaching interceptors
ROE System: PIN-protected configuration (SHA-256 hashed), three modes: MANUAL / SEMI_AUTO / AUTO, configurable confidence threshold, time threshold, engagement zone radius
→
Layer 5
Watchdog & Fault Tolerance
Supervisor: Runs as separate OS process (NOT a thread), survives crashes, OOM kills, unhandled signals. Auto-restarts AEGIS-X up to 3 times, then triggers failsafe
Failsafe Broadcaster: UDP multicast to ALL interceptors on LAN, 3 retries at 100ms spacing (<400ms total). Modes: RTB / HOLD / COMMIT_THEN_RTB. Stdlib only — works from partially crashed process
Heartbeat Monitor: Detects pipeline stalls before they become failures
